Load (tunable values, benchmark scores, status) to warm-up the optimizer.

If last_trial_id is present, load only the data from the (completed) trials that were scheduled after the given trial ID. Otherwise, return data from ALL merged-in experiments and attempt to impute the missing tunable values.

Return an iterator over PENDING Trial instances that have a scheduled start time to run on or before the specified timestamp.

Parameters:

  • timestamp (datetime.datetime) – The time in UTC to check for scheduled Trials.

  • running (bool) – If True, include the Trials that are also RUNNING or READY. Otherwise, return only the scheduled trials.

  • trial_runner_assigned (bool | None) – If True, include the Trials that are assigned to a TrialRunner. If False, return only the trials that are not assigned to any TrialRunner. If None, return all trials regardless of their assignment.
